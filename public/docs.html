App

API Documentation

This page describes the API endpoints and detection rules for the Finance Anomaly Detector.

Endpoints

GET /health

Check application status.

GET /api/expenses

Returns recent expenses.

POST /api/expenses

Create a single expense. JSON body:

{ "Amount": 100.50, "Currency": "USD", "Category": "Groceries", "Merchant": "Walmart", "Description": "Weekly groceries", "Date": "2026-05-03T10:00:00Z" }

POST /api/expenses/import-csv

Upload a CSV file with form-data 'file'. Expected columns: Date, Amount, Currency, Category, Merchant, Description.

GET /api/anomalies

Get a list of detected anomalies.

POST /api/anomalies/run

Trigger the anomaly detection engine across all expenses.

Anomaly Rules

  • High Expense: Over 3x the category average.
  • Merchant Spike: Over 4x the merchant average.
  • Category Change: Merchant used in an unusual category compared to past transactions.
  • Night Spending: Transactions made between 1 AM and 4 AM.

Example CURL

curl -X POST http://localhost:5000/api/anomalies/run
