API Documentation

All /api endpoints except login/register require an authenticated cookie session. Unauthenticated requests receive 401.

Auth

POST /api/register

Create an account and sign in. Username: 3-50 chars (letters, digits, . _ -); password: 8-128 chars. Rate limited per IP.

{ "Username": "alex", "Password": "correct-horse-battery" }

POST /api/login  /  POST /api/logout

Same body as register. Login is rate limited (10/min per IP; returns 429 beyond that).

GET /api/me

Returns the current username.

POST /api/account/change-password

{ "OldPassword": "...", "NewPassword": "..." }

Expenses

GET /api/expenses

Paginated listing. Query params: page (default 1), pageSize (default 50, max 200), category, search (merchant/description substring), from, to (dates; a bare to date is inclusive). Returns { items, total, page, pageSize }.

POST /api/expenses

{ "Amount": 100.50, "Currency": "USD", "Category": "Groceries", "Merchant": "Walmart", "Description": "Weekly groceries", "Date": "2026-05-03T10:00:00Z" }

PUT /api/expenses/{id}

Full update with the same body as POST. 404 if the expense doesn't exist or belongs to another user.

DELETE /api/expenses/{id}

Deletes the expense and any anomalies linked to it.

POST /api/expenses/import-csv

Multipart form upload (file). Headers (case-insensitive): Date, Amount, Currency, Category, Merchant, Description. Limits: 5 MB, 10,000 rows. Invalid rows are skipped and reported with row numbers.

GET /api/expenses/export

Downloads all your expenses as RFC 4180 CSV.

Anomalies

POST /api/anomalies/run

Re-evaluates every rule over your full history. Anomalies you dismissed stay dismissed (matched by expense + rule).

GET /api/anomalies

Open (unresolved) anomalies, newest first.

PATCH /api/anomalies/{id}/resolve

Dismisses an anomaly permanently.

Insights

GET /api/stats  /  /api/categories  /  /api/trends

Dashboard aggregates: headline stats, per-category totals, monthly trend series.

GET /api/reports/monthly?month=yyyy-MM

Monthly report: total, transaction count, previous-month comparison, per-category share, top 5 merchants, anomaly count. Defaults to the current month.

GET /api/recurring

Detected recurring charges: merchant, dominant category, average amount, occurrences, months active.

Budgets

GET /api/budgets  /  POST /api/budgets  /  DELETE /api/budgets/{category}

{ "Category": "Groceries", "LimitAmount": 500 }

Detection Rules

  • CAT_OUTLIER: z-score ≥ 3 against the category history (needs 5+ samples; 3x-average fallback for 3-4).
  • MERCHANT_SPIKE: over 4x the merchant's average.
  • CAT_CHANGE: merchant used in an unusual category.
  • DUPLICATE: same merchant and amount within 10 minutes; only the later charge is flagged.
  • SUB_HIKE: a stable recurring charge priced 20%+ above its usual amount.
  • NIGHT: transactions between 1 AM and 4 AM.

Example

curl -c jar -X POST localhost:5000/api/login \ -H 'Content-Type: application/json' \ -d '{"username":"alex","password":"..."}' curl -b jar -X POST localhost:5000/api/anomalies/run
